## Descripción del Proyecto

Acortador de URLs de **alto rendimiento** construido con **arquitectura serverless**, que permite convertir enlaces largos en URLs cortas personalizables con gestión completa de enlaces.

El proyecto está implementado con Vercel Edge Functions para redirecciones ultra-rápidas y Redis para almacenamiento persistente, garantizando máxima disponibilidad y velocidad.

## Características Principales

### Funcionalidades Core

Sistema completo de acortamiento de URLs con las siguientes capacidades:

- Generación de URLs cortas con slugs personalizados o aleatorios
- Redirección ultra-rápida utilizando Edge Functions (menos de 100ms)
- Códigos QR dinámicos para cada enlace acortado con descarga directa
- Gestión completa de enlaces con edición y eliminación
- Historial local por usuario sin necesidad de registro
- Interfaz moderna y responsiva con diseño minimalista

- **Persistencia** garantizada con almacenamiento en Redis (Upstash)
- **Mobile-first** con diseño responsive optimizado para dispositivos móviles

## Desafíos Técnicos Resueltos

### Desafío 1: Redirecciones Ultra-Rápidas

**Problema**: Minimizar la latencia en las redirecciones a nivel global para ofrecer la mejor experiencia de usuario.

**Solución**: Implementé Vercel Edge Functions que se ejecutan en el edge más cercano al usuario, logrando latencias menores a 100ms en cualquier parte del mundo. Esto garantiza que las redirecciones sean prácticamente instantáneas.

### Desafío 2: Gestión sin Base de Datos Tradicional

**Problema**: Gestionar sesiones de usuario sin implementar un sistema de autenticación complejo que afecte la experiencia.

**Solución**: Utilicé cookies HttpOnly seguras para almacenar el ID de sesión del usuario, permitiendo gestión completa de enlaces sin registro ni base de datos de usuarios. Esto simplifica enormemente la arquitectura.

### Desafío 3: Límites de Vercel Hobby Plan

**Problema**: Vercel Hobby Plan tiene límite de 12 serverless functions, lo que requiere optimización cuidadosa.

**Solución**: Optimicé la arquitectura para usar solo funciones esenciales, consolidando operaciones relacionadas y utilizando Edge Functions para redirecciones, quedando dentro del límite gratuito sin sacrificar funcionalidad.
